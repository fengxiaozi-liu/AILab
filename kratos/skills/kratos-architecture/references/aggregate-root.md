# Aggregate Root

## 作用范围

本文用于说明在 Kratos 项目中如何识别稳定业务对象、确认聚合根，以及如何围绕聚合根组织领域知识。

当问题属于以下场景时，应优先查看本文：

- 判断当前业务应围绕哪个对象组织
- 判断某个对象、文件、接口或领域能力归属哪个聚合根
- 判断一个结构是聚合根、其他实体、关系视图还是临时投影

## 规则

- 聚合根确认后，应在该实体上通过 `Info` / `List` 等字段表达与其他实体的关系
- 新增实体后如果该实体属于某一聚合根，则需要在对应的聚合根下建立`{Entity}Info`/`{Entity}List`字段
- 这些关系必须对应明确的实体结构
- 不要按页面、按钮、临时查询结果或阶段动作发明新的聚合根
- 同一实体上的能力、文件与命名应默认收敛到同一聚合根；若无法稳定映射到聚合根，通常说明建模未收敛
- 当前实体中如果字段是可枚举的则需要使用枚举值，不使用原始的字符串与数字逻辑
- 实体对象默认应显式补齐 `json tag`，实体对象的 `json tag` 默认使用 `snake_case`


## 什么是实体

实体是领域中的业务对象。

判断一个对象是不是实体，关键不在于它有哪些字段，而在于它是否能被持续识别为同一个对象。
即使它的属性、状态或关系发生变化，只要它的身份不变，它仍然是同一个实体。

因此，实体关注的是“它是谁”，而不只是“它的值是什么”。

实体通常具备以下特征：

- 有稳定身份，例如 `ID` 或其他可持续识别该对象的标识
- 承载自己的业务信息、状态或关系
- 会在多个业务动作之间持续存在，而不是一次请求里的临时结果
- 会被领域规则判断和消费，而不是单纯做协议透传

以下对象通常不应直接判断为实体：

- 临时查询结果
- 页面投影对象
- 纯协议请求或响应对象
- 纯关系视图对象


## 聚合根定位

1. 聚合根首先是实体  
   聚合根不是脱离实体单独存在的概念。应先识别有哪些实体，再从这些实体里判断谁是聚合根。

2. 聚合根是当前业务里最核心的那个实体  
   如果一个实体表达当前业务的核心对象，稳定承载状态、规则和关系，那么这个实体就是聚合根。

3. 不是所有实体都是聚合根  
   其他实体仍然是实体，只是当前围绕聚合根组织，用来补充表达这条业务边界里的其他信息。

4. 聚合根不是先验指定的  
   不能先定“谁是聚合根”，再反推实体；应先识别实体，再判断哪个实体是在表达当前业务的核心对象。

5. 聚合根的作用  
   - 收口稳定业务边界
   - 承载领域主语
   - 为 `repo`、`usecase`、领域对象、应用层对象、`proto` 提供统一命名中心
   - 让跨层对象在同一业务语义下保持收敛

6. 聚合根与其他实体的关系  
   其他实体仍然是实体。它们当前围绕聚合根组织，用来补充表达该业务边界中的其他信息。

## 实体字段约束

实体字段确认时，应补做一轮“可枚举字段检查”：
   - 如果字段值来自有限集合，例如状态、类型、结果、原因、阶段、动作，应优先定义为 enum 或 typed const
   - 如果字段会参与 `switch`、分支判断、状态流转、结果映射或跨层重复消费，应视为稳定值域并收敛为 enum
   - 如果字段只是第三方原始 payload、日志透传值或暂存原始响应，可以保留原始类型，但不应直接承载领域判断
   - 只要字段具备稳定值域并会被业务判断消费，就应建立对应 `enum` 或 `typed const`
   - 实体对象默认应显式补齐 `json tag`
   - 实体对象的 `json tag` 默认使用 `snake_case`

## 实体字段顺序

实体定义字段时，默认按以下顺序组织：

1. 普通字段
2. 时间类型字段
3. 关联信息 `Info` 字段
4. 关联信息 `List` 字段

## 聚合根识别与确认步骤

判断聚合根时，按以下顺序进行：

1. 先确认哪些结构是真正的实体。
2. 再判断当前业务到底在描述哪个核心对象。
3. 表达这个核心对象的实体，就是聚合根。
4. 其他实体如果围绕它组织，就说明它们当前位于这个聚合根的边界内。

## 示例

### 标准形态

```text
Account
|- 核心字段：id、status、type
|- 实体字段：AccountCollect、AccountFlowPage
|- 关系视图：FirstCheckUser、SecondCheckUser
```

### 典型实现

```go
type Account struct {
	ID                  uint32                 `json:"id"`
	Status              openenum.AccountStatus `json:"status"`
	Type                openenum.AccountType   `json:"type"`
	CreateTime          uint32                 `json:"create_time"`
	UpdateTime          uint32                 `json:"update_time"`
	FirstCheckUserInfo  *adminbiz.AdminUser    `json:"first_check_user_info"`
	SecondCheckUserInfo *adminbiz.AdminUser    `json:"second_check_user_info"`
	AccountCollectInfo  *AccountCollect        `json:"account_collect_info"`
	AccountFlowPageList []*AccountFlowPage     `json:"account_flow_page_list"`
}
```
