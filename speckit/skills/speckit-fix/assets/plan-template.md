# 修复计划模板

在 `Plan` 阶段，按以下结构写入 `specs/<feature>/issue.md`。

```md
## 修复计划

**关联问题**:
- ISSUE-001

**修复目标**:
[本次修复要恢复的正确行为]

**根因判断**:
[根据现象、证据和澄清结果得出的根因区域]
[根因分析尽可能的分析贴近我们的AGENT与SKILL]
[分析根因按照spec.md->plan.md->tasks.md->实现的链路来分析]

**影响范围**:
- [受影响的模块、文件、流程]

**修复边界**:
- [允许修改的范围]
- [明确不做的范围]

**决策分析**
- [为什么采用这种方案]
- [决策证据]
  - SKILL
  - 仓库代码
  - 上下文
  - 认知共识
  - etc...

**验证方式**:
- [测试、构建、日志、人工验证]
```

要求：

- `关联问题` 必须列出本次计划覆盖的全部未解决 `ISSUE-xxx`
- `修复边界` 必须包含至少一条明确不做项，防止修复过程扩散
- `验证方式` 必须能直接对应后续 `Implement` 阶段的验收动作
