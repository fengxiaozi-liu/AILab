# 澄清模板

在 `Specify` 或 `Clarify` 阶段发现信息不足时，按以下结构写入 `specs/<feature>/issue.md`。

```md
## 澄清记录

### ISSUE-001 [需要用户确认的关键问题]

**澄清目的**:
[说明为什么这个问题会阻塞后续 Plan 或 Tasks]

**互斥选项**:
- [ ] A. [选项 A，和其他选项互斥]
- [ ] B. [选项 B，和其他选项互斥]
- [ ] C. [如有必要再给第三个互斥选项]

**用户答案**:
[待用户填写；未填写前不得进入 Plan]

**澄清结论**:
[由 /fix 在 Clarify 阶段回填]
```

要求：

- 每个澄清问题都必须是后续规划所必需的信息，不得为了“问得更全”而泛化提问
- 互斥选项必须能帮助用户快速决策，避免开放式大段描述
- 若用户答案不在备选项内，也允许补充自由文本，但 `/fix` 仍需在 `澄清结论` 中归纳为单一结论
